Goal:
Jika seorang user banned, semua command yang dia kirim ditolak dengan pesan standar. Admin (ADMIN_USER_ID, ADMIN2_USER_ID) tetap bisa pakai command admin. Implementasi lokal memakai data/users.json yang sudah ada.

Yang harus kamu ubah/lakukan:

Tambahkan helper is_banned(user_id)
Tambahkan ke modul util kamu (tempat is_admin berada) atau buat di app/lib/ban.py jika sudah ada struktur:

python
Copy
# app/lib/ban.py
from app.storage_local import load_users

def is_banned(user_id) -> bool:
    if user_id is None:
        return False
    users = load_users()
    rec = users.get(str(user_id))
    return bool(rec and rec.get("banned", False))

from functools import wraps

from aiogram import types

def require_not_banned(handler):
    @wraps(handler)
    async def wrapper(message: types.Message, *args, **kwargs):
        uid = message.from_user.id if message and message.from_user else None
        if is_admin(uid):
            return await handler(message, *args, **kwargs)
        if is_banned(uid):
            await message.reply("⛔ Akun kamu diblokir dari penggunaan command.")
            return
        return await handler(message, *args, **kwargs)
    return wrapper

Admin commands tetap jalan meski pengirim banned
Jangan pasang require_not_banned di handler admin /ban, /add_premium, /remove_premium, /grant_credits. Cukup cek admin seperti yang sudah kamu punya (is_admin(...)). Ini memungkinkan admin membatalkan ban (unban via premium atau bikin command /unban nanti kalau dibutuhkan).

Edge cases yang harus dipastikan:

Saat /ban <userid> dijalankan, user target otomatis dibuat (jika belum ada) dan banned=true. Setelah itu, semua command dari user target ditolak oleh guard.

Pesan standar saat ditolak: ⛔ Akun kamu diblokir dari penggunaan command.

Admin tetap bisa menjalankan command walau dirinya sendiri flagged banned (bypass admin).
