🚀 PROMPT REPLIT AI AGENT — Fix /admin Parse Error (HTML + Chunking)

GOAL

Hilangkan error Telegram: Can't parse entities: can't find end of the entity... saat /admin.

Kirim teks /admin sebagai HTML yang sudah di-escape (bukan Markdown).

Chunk pesan panjang (<4096 chars).

Tetapkan timestamp tetap UTC (jangan ubah kode UTC yang sudah ada).

Jangan ganggu logic fitur lain.

TASKS

Buat util HTML-escape & chunking.

Update handler /admin (aiogram atau telebot — terapkan sesuai framework yang ditemukan).

Pastikan handler /admin force parse_mode="HTML" dan disable_web_page_preview=True, meski global default parse mode Markdown.

1) Tambah file util: app/utils/safe_text.py
# app/utils/safe_text.py
import html
from typing import Iterable

MAX_LEN = 3500  # aman di bawah limit 4096

def to_html(text: str) -> str:
    # Escape semua karakter HTML berbahaya agar aman diparse Telegram
    return html.escape(text, quote=False)

def chunk(text: str, max_len: int = MAX_LEN) -> Iterable[str]:
    if len(text) <= max_len:
        yield text
        return
    buf = []
    cur = 0
    for line in text.splitlines(True):  # keep newline
        if cur + len(line) > max_len and buf:
            yield "".join(buf)
            buf = [line]
            cur = len(line)
        else:
            buf.append(line)
            cur += len(line)
    if buf:
        yield "".join(buf)

2) Patch handler /admin
Jika proyek aiogram v3 (punya app/routers/admin_router.py atau serupa):

Ganti handler /admin jadi:

# app/routers/admin_router.py
from aiogram import Router, types
from app.admin import get_admin_panel_text
from app.utils.safe_text import to_html, chunk

router = Router()

@router.message(commands={"admin","status"})
async def admin_panel(msg: types.Message):
    text = get_admin_panel_text()
    html_text = to_html(text)
    for part in chunk(html_text):
        await msg.answer(
            part,
            parse_mode="HTML",            # PENTING: jangan Markdown
            disable_web_page_preview=True
        )


Jika handler /admin ada di file lain, terapkan patch yang sama di sana.

Jika proyek pyTelegramBotAPI / telebot:

Ganti handler /admin jadi:

# di file bot utama tempat handler /admin berada
from app.admin import get_admin_panel_text
from app.utils.safe_text import to_html, chunk

@bot.message_handler(commands=['admin','status'])
def cmd_admin(message):
    text = get_admin_panel_text()
    html_text = to_html(text)
    for part in chunk(html_text):
        bot.send_message(
            message.chat.id,
            part,
            parse_mode="HTML",            # PENTING: jangan Markdown
            disable_web_page_preview=True
        )

3) Override global parse mode (kalau ada)

Cari inisialisasi bot yang memberi default parse_mode="Markdown"/"MarkdownV2".

Jangan ubah default globalnya, cukup pastikan /admin mengirim dengan parse_mode="HTML" seperti di atas.

4) Jangan ubah UTC

Jangan ubah fungsi UTC yang sudah ada (mis. build_system_status() yang memformat "%H:%M:%S UTC").

Hanya pastikan output /admin dikirim dalam HTML-escaped + chunking.

ACCEPTANCE

Jalankan /admin → tidak ada lagi error “Can’t parse entities”.

Jika teks panjang (banyak diagnostic JSON/LOG), pesan otomatis terpecah menjadi beberapa bagian.

“Last Update” tetap tampil UTC.

Fitur lain tidak berubah.

Catatan debug cepat:
Kalau masih error, berarti ada handler /admin lain yang belum dipatch. Cari semua referensi commands=["admin" atau commands={"admin" dan pastikan seluruhnya memakai to_html() + chunk() + parse_mode="HTML".
