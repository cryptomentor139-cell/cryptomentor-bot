PROMPT REPLIT AI AGENT — MATIKAN AUTO SIGNALS SEMENTARA
TUJUAN

Nonaktifkan semua auto signals (job terjadwal, broadcast otomatis).

Tambah feature flag supaya gampang ON/OFF.

Sediakan command admin-only untuk cek status & memaksa OFF.

Pastikan tidak ada job yang tetap jalan saat bot start.

1) Update config: tambah flag

Edit app/config.py dan tambahkan:

import os

# ... existing config ...

# Matikan auto signals secara default (bisa override via Secrets)
AUTO_SIGNALS_ENABLED = os.getenv("AUTO_SIGNALS_ENABLED", "false").lower() in ("1","true","yes")


Di Replit Secrets, set AUTO_SIGNALS_ENABLED = false untuk memastikan OFF saat start.

2) Modul auto signals (guarded)

Buat file app/services/auto_signal.py:

from telegram.ext import ContextTypes
from app.config import AUTO_SIGNALS_ENABLED

JOB_NAME = "auto_signals_job"

async def auto_signal_tick(context: ContextTypes.DEFAULT_TYPE):
    # Jika ada panggilan residual, guard lagi di runtime
    if not AUTO_SIGNALS_ENABLED:
        return
    chat_id = context.job.chat_id if context.job else None
    if not chat_id:
        return
    # Taruh logic auto signal lama kamu di sini jika nanti diaktifkan
    # await context.bot.send_message(chat_id, "Auto-signal tick (demo)")
    return

def schedule_auto_signals(job_queue, chat_id: int, interval_seconds: int = 900):
    """Dipanggil saat startup untuk mendaftarkan job jika diizinkan."""
    # Pastikan tidak dobel
    cancel_auto_signals(job_queue)
    if not AUTO_SIGNALS_ENABLED:
        return  # dihormati: OFF
    # Jadwalkan job periodik (contoh: tiap 15 menit)
    job_queue.run_repeating(
        auto_signal_tick,
        interval=interval_seconds,
        first=10,  # mulai 10 detik setelah start
        name=JOB_NAME,
        chat_id=chat_id,
    )

def cancel_auto_signals(job_queue):
    jobs = job_queue.get_jobs_by_name(JOB_NAME)
    for j in jobs:
        j.schedule_removal()

3) Hook di entry bot

Edit app/bot.py:

Tambahkan import dan pemanggilan scheduler saat build app, tapi hormati flag.

# Tambah import di atas
from app.config import AUTO_SIGNALS_ENABLED
from app.services.auto_signal import schedule_auto_signals, cancel_auto_signals

# Di build_application(), setelah app dibuat:
def build_application() -> Application:
    # ... existing code ...
    app = Application.builder().token(BOT_TOKEN).build()

    # ... handlers ...

    # Jadwalkan auto signals HANYA jika enabled (dan kamu punya default chat id tujuan).
    # Jika auto signals broadcast ke group tertentu, isi chat_id target di Secrets atau hardcode sementara.
    target_chat_id = None  # ganti jika perlu
    if target_chat_id:
        schedule_auto_signals(app.job_queue, chat_id=target_chat_id)

    return app


Karena sekarang kamu ingin OFF, biarkan AUTO_SIGNALS_ENABLED=false dan/atau jangan set target_chat_id.

4) Command admin untuk kontrol & status

Masih di app/bot.py, tambahkan handler:

from app.auth.access import admin_only
from app.config import AUTO_SIGNALS_ENABLED

@admin_only
async def cmd_autosignal_status(update: Update, context: ContextTypes.DEFAULT_TYPE):
    jobs = context.application.job_queue.get_jobs_by_name("auto_signals_job")
    running = "✅ aktif" if jobs else "❌ tidak aktif"
    enabled_flag = "ON" if AUTO_SIGNALS_ENABLED else "OFF"
    await update.effective_message.reply_text(
        f"🛰️ Auto Signals Status\n- Flag: *{enabled_flag}*\n- JobQueue: {running}",
        parse_mode=ParseMode.MARKDOWN
    )

@admin_only
async def cmd_autosignal_off(update: Update, context: ContextTypes.DEFAULT_TYPE):
    cancel_auto_signals(context.application.job_queue)
    await update.effective_message.reply_text(
        "⏸️ Auto signals *dimatikan* untuk sesi ini.\n"
        "_Catatan: setting ini tidak persisten. Set `AUTO_SIGNALS_ENABLED=false` di Secrets agar OFF saat restart._",
        parse_mode=ParseMode.MARKDOWN
    )

# (Opsional) aktifkan kembali manual bila nanti diperlukan
@admin_only
async def cmd_autosignal_on(update: Update, context: ContextTypes.DEFAULT_TYPE):
    # Wajib punya target chat id; contoh gunakan chat saat ini
    chat_id = update.effective_chat.id
    schedule_auto_signals(context.application.job_queue, chat_id=chat_id)
    await update.effective_message.reply_text(
        "▶️ Auto signals *diaktifkan* untuk sesi ini.",
        parse_mode=ParseMode.MARKDOWN
    )

# Registrasikan command
app.add_handler(CommandHandler("autosignal_status", cmd_autosignal_status))
app.add_handler(CommandHandler("autosignal_off", cmd_autosignal_off))
app.add_handler(CommandHandler("autosignal_on", cmd_autosignal_on))  # opsional


Dengan ini:

/autosignal_status → lihat flag & apakah job sedang terdaftar.

/autosignal_off → batalkan semua job yang aktif sekarang.

(Opsional) /autosignal_on → aktifkan sementara (tidak persisten).

5) Hard kill semua jejak auto-scheduler lama

Cari di project:

run_repeating(, run_once(, run_daily(, apscheduler, schedule, asyncio.create_task(auto_signal), atau loop manual yang mem-broadcast sinyal.

Ganti setiap pemanggilan jadi melalui schedule_auto_signals(); atau comment out seluruh scheduling lama.

6) Non-code checklist (penting!)

Replit Secrets: set AUTO_SIGNALS_ENABLED = false.

Cron eksternal / Uptime pinger / GitHub Actions / Replit Scheduled Runs: pastikan dimatikan kalau ada yang memicu endpoint auto signals/broadcast.
