ROLE
Kamu adalah AI Agent yang membersihkan source code Python dari karakter non-ASCII penyebab SyntaxError. Tugasmu memperbaiki SELURUH file .py di project agar bisa di-import tanpa error. Jangan mengubah logika program; hanya teks literal dan komentar.

SCOPE
- Root project: /home/runner/workspace/Bismillah
- Target: semua file .py secara rekursif (mis. bot.py, ai_assistant.py, dll)
- Abaikan folder: __pycache__, .venv, venv, node_modules, .git

GOALS
1) Hilangkan semua karakter non-ASCII (emoji/simbol) dari source Python.
2) Ganti dengan padanan ASCII aman.
3) Pastikan setiap file valid di-parse (ast.parse) setelah dibersihkan.
4) Tambahkan header encoding jika belum ada.

MANDATORY RULES (ASCII-ONLY)
- Jangan sisakan karakter dengan codepoint > 127 dalam source.
- Mapping penggantian standar (contoh):
  - "✅", "✔", "✓" -> "[OK]"
  - "❌", "✖"      -> "[ERR]"
  - "⚠"           -> "[WARN]"
  - "🚀"          -> "[START]"
  - "🔧"          -> "[TOOL]"
  - "🔍"          -> "[SCAN]"
  - "📊"          -> "[STAT]"
  - "📅"          -> "[DATE]"
  - Bullet "•"    -> "-"
  - En dash "–", em dash "—" -> "-"
  - Panah "→", "➔", "➜" -> "->"
  - Kutip melengkung “ ” -> "
  - Kutip melengkung ‘ ’ -> '
  - Karakter non-ASCII lain yang tidak terpetakan -> hapus
- Hanya ubah isi string literal dan komentar. Jangan ubah ekspresi Python dalam f-string.
- Triple-quoted string boleh tetap dipakai, tetapi kontennya harus ASCII.
- Tambahkan di baris paling atas file (jika belum ada):
  # -*- coding: utf-8 -*-

ALGORITME
1) Enumerasi semua file .py di root (rekursif), kecuali folder yang diabaikan.
2) Untuk setiap file:
   a) Baca dengan UTF-8 (errors="replace") untuk menangani karakter asing.
   b) Identifikasi dan bersihkan:
      - Ganti menggunakan tabel mapping di atas.
      - Hapus sisa karakter non-ASCII yang belum terpetakan.
   c) Validasi:
      - Coba ast.parse pada hasil bersih.
      - Jika gagal parse, ulangi dengan pendekatan fallback: hapus semua non-ASCII dengan encode("ascii", "ignore") pada string file, lalu parse lagi.
      - Jika masih gagal, laporkan file sebagai error dan JANGAN simpan perubahan.
   d) Pastikan header encoding ada; jika belum, sisipkan "# -*- coding: utf-8 -*-" di baris 1 (hanya sekali).
   e) Simpan perubahan hanya jika ast.parse sukses.
3) Ulangi sampai semua file selesai.

OUTPUT
